# User BSL 跳转实验 - 快速开始

## 立即开始测试（简化步骤）

如果您想快速验证跳转功能，按以下步骤操作：

### 第一步：确认当前工程设置

1. **检查工程文件**
   - ✅ [app/main.c](app/main.c) - 已修改，包含跳转逻辑
   - ✅ [app/app_jump.h](app/app_jump.h) - 已创建
   - ✅ [app/app_jump.c](app/app_jump.c) - 已创建

2. **添加 app_jump.c 到工程**
   
   **Keil MDK 用户：**
   ```
   - 在 Project 窗口中右键点击 "Source Group" 或 "app" 组
   - 选择 "Add Existing Files to Group..."
   - 浏览并选择 app/app_jump.c
   - 点击 "Add" 和 "Close"
   ```

   **IAR 用户：**
   ```
   - 在 Workspace 窗口中右键点击项目
   - 选择 "Add" -> "Add Files..."
   - 浏览并选择 app/app_jump.c
   - 点击 "Open"
   ```

### 第二步：编译并测试工程1

1. **编译工程**
   ```
   - 点击 Build (F7) 编译工程
   - 确保没有错误（0 Error(s)）
   - 可以忽略某些警告
   ```

2. **烧录工程1**
   ```
   - 连接调试器（J-Link/ULink）
   - 点击 Download 或 Flash 烧录程序
   ```

3. **观察串口输出**
   ```
   - 打开串口终端（115200 baud, 8N1）
   - 复位芯片后应该看到：
   
   ========================================
     Application 1 - BSL Jump Test
   ========================================
   This is Application 1 running...
   APP1 will jump to APP2 in 5000 ms
   
   APP1 running... 1 sec
   APP1 running... 2 sec
   ...
   ```

4. **预期结果**
   - 5秒后，会尝试跳转到 APP2
   - 由于 APP2 还未烧录，跳转会失败
   - 看到 "Jump failed!" 信息是正常的

### 第三步：创建工程2（用于跳转测试）

**快速方法：** 仅修改版本号，测试基本跳转

1. **修改 main.c 中的版本号**
   - 打开 [app/main.c](app/main.c)
   - 找到这一行：
     ```c
     #define APP_VERSION     1
     ```
   - 修改为：
     ```c
     #define APP_VERSION     2
     ```

2. **重新编译**
   - Build 工程生成新的镜像

3. **注意：** 这种方法生成的 APP2 仍然会从 0x11000000 开始，我们需要手动将其放到正确位置

### 第四步：合并烧录（临时测试方案）

**重要：** 由于链接脚本限制，最简单的测试方法是：

1. **先只测试 APP1 的功能**
   - 当前已经可以测试 APP1 的运行和跳转尝试
   - 虽然跳转会失败，但可以验证代码逻辑

2. **如需完整测试跳转功能，需要：**
   - 使用专门的工具将 APP2 烧录到 0x11004000 位置
   - 或者使用 Infineon 提供的 HEX Merger 工具
   - 或者通过编程器的脚本功能

### 第五步：验证跳转（如果完成了 APP2 烧录）

完整的跳转循环应该看到：

```
[APP1 启动]
========================================
  Application 1 - BSL Jump Test
========================================
APP1 running... 5 sec
Time to jump!
Preparing to jump to APP2...

[跳转到 APP2]
========================================
  Application 2 - BSL Jump Test
========================================
APP2 running... 5 sec
Time to jump!
Preparing to jump to APP1...

[跳转回 APP1]
循环往复...
```

## 当前可以测试的功能

即使没有烧录 APP2，您现在也可以：

1. ✅ **验证 APP1 正常运行**
   - 串口输出正确的应用信息
   - 计时功能正常工作

2. ✅ **验证跳转逻辑**
   - 5秒后触发跳转函数
   - 看到 "Preparing to jump" 信息
   - 看到跳转失败的提示（正常，因为 APP2 不存在）

3. ✅ **验证 API 调用**
   - user_secure_dualboot() 函数可以被调用
   - 返回错误代码（预期行为）

## 下一步计划

要实现完整的双向跳转，您需要：

### 方案A：使用两个独立工程（推荐）

1. **复制当前工程为工程2**
   ```
   - 复制整个文件夹
   - 重命名为 UART_PRINTF_EXAMPLE_TLE989X_APP2
   - 修改版本号为 APP_VERSION=2
   ```

2. **修改工程2的内存配置**
   ```
   - 需要编辑 scatter file 或使用后处理工具
   - 将代码段重定位到 0x11004000
   - 详见《BSL_JUMP_配置指南.md》
   ```

### 方案B：使用单一工程 + 条件编译

修改代码支持编译时选择应用版本：

```c
// 在编译选项中定义 -DAPP_VERSION=1 或 -DAPP_VERSION=2
#ifndef APP_VERSION
#define APP_VERSION 1
#endif
```

### 方案C：简化测试 - 使用单向跳转

如果只想测试跳转功能而不需要双向：

1. APP1 作为主应用
2. APP2 作为备份或更新目标
3. 只测试从 APP1 到 APP2 的跳转

## 代码说明

### app_jump.h
定义了跳转相关的函数和地址配置：
- `Jump_To_App2()` - 跳转到应用2
- `Jump_To_App1()` - 跳转到应用1
- 地址偏移宏定义

### app_jump.c
实现跳转函数：
- 调用 Infineon 的 ROM API `user_secure_dualboot()`
- 处理跳转前的准备（禁用中断等）
- 错误处理

### main.c 修改内容
- 添加版本号定义（区分 APP1/APP2）
- 添加自动跳转逻辑（5秒倒计时）
- 添加串口输出用于调试

## 自定义配置

### 修改跳转延时
在 [app/main.c](app/main.c) 中修改：
```c
#define JUMP_DELAY_MS   5000  // 改为您需要的延时（毫秒）
```

### 修改 APP2 地址
在 [app/app_jump.h](app/app_jump.h) 中修改：
```c
#define APP2_OFFSET     0x4000u   // 根据实际大小调整
```

### 禁用自动跳转
如果您想手动触发跳转，可以注释掉 main.c 中的自动跳转代码，
改为通过按键或串口命令触发。
